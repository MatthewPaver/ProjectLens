Transparent by design

How the model works

  1. Describe risk drivers. Each driver has a probability, triangular impact range and critical-path exposure.
  2. Run 10,000 futures. The seeded simulation produces P10, P50, P80 and on-time confidence.
  3. Apply an intervention. A scenario reduces named driver impacts. Baseline and scenario use identical random draws.
  4. Rank the evidence. Root causes use probability × most-likely impact × criticality.
  5. Record the outcome. Selected interventions enter the decision ledger for review against the next update.

This is an explainable decision-support demonstrator using synthetic data, not a certified schedule-risk model.
